Syntax documentation


  • Variables

    We can use 3 different variable notations - only 2 of them being mutable options.

    • var

      This is a standard global variable notation. It can be referenced from anywhere, and is therefore not an ideal way of declaring variables.

    • let

      The let notation is used for declaring variables inside a scope.

      This means that the variable cannot be referenced in other scope or global namespace.

    • const

      This notation states that the variable holds the value it is initiated with. It is therefore non-mutable.

      A constant doesn't need the overhead memory that a standard variable needs, because the system knows that the value will never change.


  • Strings

    Strings are pieces of text that can be worked with in different ways. They can contain all characters, as the characters, no matter their function, are turned into text.

    • Comma separated sentences

      Strings can be built as variables or in the console.

      This way, javascript will insert spaces on it's own after the comma.

      let firstName = "Thomas"
      let lastName = "Vindelev"

      console.log("My first name is", firstName, "and my last name is", lastName)

      // My first name is Thomas and my last name is Vindelev

    • Template strings

      By using backticks (  `  ) as string identifiers, we can inject variables into a string of text.

      let firstName = "Thomas"
      let lastName = "Vindelev"
      let myName = `My name is ${firstName}`

      console.log(myName)
      console.log(`My first name is ${firstName} and my last name is ${lastName}`)

      // My name is Thomas
      // My first name is Thomas and my last name is Vindelev

    • Parsing numbers from strings

      Using the parse methods we can cast and work with numerical values from strings.

      However, we can only use these methods if the string only contains numbers and not letters.

      Parsing numbers from strings including text will result in NaN (not a number).

      var numberOne = "1.10"
      var numberTwo = "2.30"
      console.log(parseFloat(numberOne) + parseFloat(numberTwo))

      // 3.4

  • Objects

    Objects can store key-value pairs. These can contain regular data types such as strings or numbers, but they can also contain functions.

    • Javascript objects

      Objects in Javascript can be declared with or without key-value pairs.

      In this example an object is created holding my name and age.

      let javascriptObject = {
          name: "Thomas Skovgaard Vindelev",
          age: 24
      }

    • JSON objects

      Javascript objects and JSON objects look very similar, but the difference is that JSON needs every key to have quotation marks.

      Data types such as integers or floats don't need to be annotated with quotation marks, since they're numerical values.

      Because JSON is used so frequently in combination with Javascript we often just use the JSON way of creating objects.

      let jsonObject = {
          "name": "Thomas Skovgaard Vindelev",
          "age": 24
      }

    • Accessing objects

      Adding, modifying or deleting values in an object is possible in several ways.

      We can add values at the creation of the object, or we can add them later.

      In this example I made the empty object myObject. Then I gave it a key named newItem with a value of 10, then I modified it and deleted it.

      let myObject = {}
      myObject["newItem"] = 10
      myObject.newItem = 20
      delete myObject.newItem
