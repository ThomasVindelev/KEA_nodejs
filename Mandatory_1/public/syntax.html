Syntax documentation

Javascript syntax is not that different from Java, however it's very different in terms of what we're allowed to do and how we can do it.

For starters we are very used to the idea of working with different classes as objects.

  • Variables

    We can use 3 different variable notations - only 2 of them being mutable options.

    • var

      This is a standard global variable notation. It can be referenced from anywhere, and is therefore not an ideal way of declaring variables.

    • let

      The let notation is used for declaring variables inside a scope.

      This means that the variable cannot be referenced in other scope or global namespace.

    • const

      This notation states that the variable holds the value it is initiated with. It is therefore non-mutable.

      A constant doesn't need the overhead memory that a standard variable needs, because the system knows that the value will never change.


  • Strings

    Strings are pieces of text that can be worked with in different ways. They can contain all characters, as the characters, no matter their function, are turned into text.

    • Comma separated sentences

      Strings can be built as variables or in the console.

      This way, javascript will insert spaces on it's own after the comma.

      let firstName = "Thomas"
      let lastName = "Vindelev"

      console.log("My first name is", firstName, "and my last name is", lastName)

      // My first name is Thomas and my last name is Vindelev

    • Template strings

      By using backticks (  `  ) as string identifiers, we can inject variables into a string of text.

      let firstName = "Thomas"
      let lastName = "Vindelev"
      let myName = `My name is ${firstName}`

      console.log(myName)
      console.log(`My first name is ${firstName} and my last name is ${lastName}`)

      // My name is Thomas
      // My first name is Thomas and my last name is Vindelev

    • Parsing numbers from strings

      Using the parse methods we can cast and work with numerical values from strings.

      However, we can only use these methods if the string only contains numbers and not letters.

      Parsing numbers from strings including text will result in NaN (not a number).

      let numberOne = "1.10"
      let numberTwo = "2.30"
      console.log(parseFloat(numberOne) + parseFloat(numberTwo))

      // 3.4

  • Objects

    Objects can store key-value pairs. These can contain regular data types such as strings or numbers, but they can also contain functions.

    • Javascript objects

      Objects in Javascript can be declared with or without key-value pairs.

      In this example an object is created holding my name and age.

      let javascriptObject = {
          name: "Thomas Skovgaard Vindelev",
          age: 24
      }

    • JSON objects

      Javascript objects and JSON objects look very similar, but the difference is that JSON needs every key to have quotation marks.

      Data types such as integers or floats don't need to be annotated with quotation marks, since they're numerical values.

      Because JSON is used so frequently in combination with Javascript we often just use the JSON way of creating objects.

      let jsonObject = {
          "name": "Thomas Skovgaard Vindelev",
          "age": 24
      }

    • Accessing objects

      Adding, modifying or deleting values in an object is possible in several ways.

      We can add values at the creation of the object, or we can add them later.

      In this example I made the empty object myObject. Then I gave it a key named newItem with a value of 10, then I modified it and deleted it.

      let myObject = {}
      myObject["newItem"] = 10
      myObject.newItem = 20
      delete myObject.newItem

  • Functions

    Functions are used to execute a number of tasks when called. Functions are also objects, as variables can be assigned at functions.

    Furthermore, functions can be passed into functions as callback functions.

    • Callback functions

      A callback function is injected into another functions arguments. These methods can also be defined directly in the arguments.

      Because functions are objects we can parse different methods into other methods, in which they will be executed when the initial function tells it to.

      The callbacks can be either synchronous or asynchronous, depending on when the callback function will execute.

  • Loops

    Loops can be used to run a block of code several times with, sometimes, different outcomes.

    The most common loops in Javascript are for-loops and while-loops

    • For-loop

      A for-loop is characterized by the keyword for in front of the statement itself.

      The loop often relies on an incrementer, but can also be done as a for/in-loop.

      for-loop

      for (i = 0; i < 2; i++) {
          console.log("Hello, number",i)
      }

      // Hello, number 0
      // Hello, number 1
      for/in-loop

      let me = {firstName: "Thomas", lastName: "Vindelev"}

      for (names in me) {
          console.log(names)
      }

      // Thomas
      // Vindelev

    For-loops in Javascript are similar to the ones we find in Java.

    However we don't need to declare i as an integer, as Java and Javascript don't have the same rules regarding data types.


  • Arrays

    Arrays are collections that can hold different values. Different data types can be stored in the same array.

    Javascript arrays has a bunch of inherited methods that can be used to achieve different results.

    • Index

      To access data in an array we need to specify the index in which the desired data is located.

      Since we can also store objects in an array, can parse a function from an array.

      function calculate(first, second, operation) {
          return operation(first, second)
      }

      function add(first, second) {
          return first + second
      }

      function subtract(first, second) {
          return first - second
      }

      let operations = [add, subtract]

      console.log(calculate(2, 3, operations[0])) // operations returns the first method
      console.log(calculate(2, 3, operations[1])) // operations returns the second method

      // 5
      // -1

      We can also find the position of a specific element bu using the indexOf()-method, which comes with the array.

      let anArray = ["First element", "Second element"]
      console.log(anArray.indexOf("First element"))

      // 0

    • Push

      This enables you to add, or rather push, new elements to an already existing array.

      To print an array in a way we can easily read in the console we can use the table function in console.

      let newArray = []
      newArray.push(me = {firstName: "Thomas", lastName: "Vindelev"}, teacher = {firstName: "Anders", lastName: "Latif"})
      console.table(newArray)

    • Pop

      Pop will remove the last element in an array and return it. This can be useful if you want

      let cars = ["Mercedes", "BMW", "Audi"]
      console.table(cars)



      cars.pop
      console.table(cars)

    • Slice

      Slice will copy part of or an entire array to another array. This can be changed by the parameters given to the method.

      In the next example, the newCars array will have all cars from the first index and forward from the cars array.

      let newCars = cars.slice(1)
      console.table(newCars)

    • Spilce

      If you want to remove part of an array, then you can use the spilce method.

      Other elements can be placed into the array instead, and this will push other elements forward.

      let cars = ["Mercedes", "BMW", "Audi"]
      cars.splice(cars.indexOf("BMW"), 0, "Toyota", "Honda", "Jaguar")
      console.table(cars)

    • For each

      A for each loop do a certain task for each element in an array.

      These loops can be made in different ways. This example demonstrates the forEach-method.

      let numbers = [1, 2, 3]

      numbers.forEach(number => {
          console.log(number)
      })

      // 1
      // 2
      // 3
